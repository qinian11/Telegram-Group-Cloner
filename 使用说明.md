# 使用说明

## 1. 安装与启动

```powershell
python -m pip install -r requirements.txt
python main.py
```

程序首次启动会自动创建 `setting`、`sessions`、`sessions_banned`、`cache`、`logs`、`exports` 目录。
默认窗口按最小可操作尺寸 `960×640` 打开，运行日志默认显示在底部栏。可在“设置”中切换右侧栏或底部栏。

## 2. 首次配置流程

1. “群组配置”已内置 Telegram Desktop 官方 API，只需填写监听手机号；监听账号 session 会自动使用手机号命名。
2. 如需代理，启用并填写 socks5、socks4 或 http 代理信息。
3. 填写源群和目标群，源群支持逗号或换行批量输入。
4. 配置黑名单、关键词和替换规则。普通文本替换支持 `old->new`，也兼容 `old-new`，例如 `5-4` 表示把 `5` 替换成 `4`。
5. 点击“保存配置”。

## 3. 登录账号

在“账号池”点击“登录”：

- 手机验证码登录：输入手机号，点击“获取验证码”。session 文件会自动按手机号命名；验证码输入框支持 `resend` 重新请求，支持 `sms` 尝试短信通道。
- 二级验证密码：如果 Telegram 账号开启两步验证，请在“二级验证密码”中填写 Telegram 云密码。
- 扫码登录：点击“生成/刷新二维码”后，窗口内会直接显示可扫描二维码。用手机 Telegram 进入“设置 > 设备 > 连接桌面设备”扫描二维码。登录成功后 session 会自动改成账号手机号，下方链接仅作为备用复制方式。
- 扫码超时或关闭登录窗口时，程序会主动断开后台扫码登录并清理临时 `qr_pending.session`，避免临时 session 被占用导致无法删除。
- 批量导入 session：可一次选择多个 `.session` 文件；同名 `.json` 元数据会一并导入。

程序默认内置并使用 Telegram Desktop 官方 API，导入 session 后也会优先使用该 API 检测授权：

```text
api_id: 2040
api_hash: b18441a1ff607e10a989891a5462e627
```

如果某个 API 候选返回未授权或连接失败，程序会继续尝试后续候选；只有所有候选均不可用时才标记为未授权或连接失败。

## 4. 登录窗口找回

“登录账号”窗口现在是可找回的普通窗口：

- 可以用 `Alt+Tab` 找回“登录账号”窗口。
- 再次点击主界面的“登录”按钮，会把已经打开的登录窗口拉回前台。
- 登录窗口支持 `Esc` 关闭。

## 5. 启动监听

1. 确保监听账号 session 存在且授权可用。
2. 确保至少一个克隆账号 session 可用。
3. 点击顶部“校验配置”。
4. 校验通过后点击“开始监听”。
5. 运行中同一个按钮会变为“停止监听”。

消息转发、编辑同步和删除同步都会让克隆账号使用自己的目标群标识发送，不复用监听账号解析出来的群实体，避免监听账号可访问但克隆账号发送时报“群组不可访问”。

监听源群消息时会跳过机器人账号发出的消息，只克隆普通用户发言。

消息转发成功后会在 `cache/message_map.json` 保存源消息与目标消息映射，用于回复关系、编辑同步和删除同步。转发失败不会写入成功映射。

如果源群和目标群配置成同一个群，程序会跳过克隆账号自己发出的消息，避免转发成功后再次被监听到形成循环转发。
